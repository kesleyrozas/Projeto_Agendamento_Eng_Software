<Repositorio Projeto Agendamento SysMédico>

Requisitos

-- Sistema de Agendamento Médico

-- Visual – O software terá telas dinâmicas, responsivas e poderá ser acessado pelo celular,
contara com telas com informações claras e objetivas.

-- Funcional – O sistema irá funcionar em nuvem para que o médico consiga acesso de fora do
hospital.

-- Estrutural –
	- Tela de Login com logo da empresa.
	- O sistema contará com tela de cadastro dos clientes e ficara visível para consulta.
	- Após realizar o cadastro do cliente o sistema irá abrir uma tela para fazer o agendamento das
pacientes e vincular ao médico de atendimento, ao fazer o cadastro vai ser aberto uma modal
para agendar o cliente na data e horário disponível.
	- O sistema notificará os clientes nos últimos 3 dias antes da consulta para ser recordar.
	- Ao Chegar para consulta o atendente irá lançar no sistema de agendamento um checkin para
confirmar o atendimento do paciente.
	- Caso haja retorno para ser marcado o médico operante irá marcar quantos dias são
necessários para próxima consulta e o sistema irá encaixa o paciente em um horário
automático, sendo aprovado pelo paciente será enviado uma notificação para o e-mail com a
nova data da consulta.
	- O sistema contará com tela para verificar quais dias da semana tem consulta e terá disponível
um relatório mensal dos atendimentos realizados.
	- A cada paciente vinculado com o médico, ele será notificado via sms sobre.
	- Pacientes terão a possibilidade fazer uma solicitação de consulta online pelo software, que
será aprovado ou não pelos operantes do sistema.
